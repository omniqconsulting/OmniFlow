2

Set Up Checklists

SA Library → create template → add steps → deploy to tenant
Go to Library →
Step by step
①
Go to Library → Checklists tab. Click "+ New Checklist Template".
②
Give it a name, optional description, choose frequency (Daily / Weekly / Monthly / Ad-hoc) and set due hour if daily.
③
Add Steps (checklist items) in order. Each step has a title and optional instruction text. Drag to reorder.
④
Click "Deploy to Tenant". Select the tenant from the dropdown. Done — it will appear in that tenant's Checklists module.
⑤
The tenant ADMIN assigns it to employees. They complete it from their dashboard daily.

3

Set Up the Flow Board (FMS)

Library → create flow → add stages → attach sub-modules → deploy
Go to Library →
Step by step
①
Go to Library → Flows tab. Click "+ New Flow". Give it a name and description.
②
Add Stages in pipeline order. Each stage is a column on the Flow Board kanban. Example: Quote → Design → Production → QC → Dispatch → Closed.
③
For each stage that needs a sub-action, click "+ Sub-module" and attach:
  • MATERIAL_REQ — triggers a material request to the Store Manager when a ticket enters this stage
  • DISPATCH — triggers a dispatch form at the Dispatch stage
  • INVOICE — triggers invoice generation
④
Click "Deploy to Tenant". Select the tenant. Done — the pipeline appears in their Flow Board.
⑤
The ADMIN or MANAGER creates the first ticket and assigns it. The ticket moves through stages as work progresses. Sub-modules trigger automatically.

How it works at Acme:
1. MANAGER creates "Production Order #PO-001" ticket.
2. Ticket moves to Production → PMS sub-module tracks daily progress.
3. Ticket moves to Dispatch → DISPATCH form fires → driver details and vehicle logged.
4. Ticket closes → auto-KPI updates on all dashboards.
Role coverage for this flow
MANAGER — creates + moves tickets
EMPLOYEE — assigned to production/QC stages
ADMIN — sees all, can reassign
Sub-module tip: PMS on the Production stage tracks daily progress. DISPATCH on the Dispatch stage captures driver details and vehicle. Both sub-modules fire as soon as the ticket enters the stage.

How the admin adds employees
1. Log in as ADMIN → go to Employees
2. Click "+ Add Employee"
3. Fill name, phone number, role, password
4. Employee logs in with: Factory ID + phone + password
Remind the admin: EMPLOYEE is a restricted role — they only see their assigned tickets and checklists. They do not see the full admin or manager dashboard. This is intentional for domain separation.

What happens after go-live

• Library templates can be updated and re-deployed

⚡ Quick Reference

Tenant creation order
Create Tenant → Add Modules → Create Admin → Deploy Library items → Add Employees
Module gating rules
Tickets = always on. Everything else = SA opts in per tenant. Override plan defaults via Feature Flags on tenant page.
Sub-module trigger timing
Fires when a ticket enters the stage. PMS, Dispatch, and Invoice sub-modules activate automatically. Custom sub-modules render a form the assignee fills in.
Changing modules after creation
Tenant detail page → Feature Flags section → toggle any module on or off. Takes effect immediately — no restart needed.
